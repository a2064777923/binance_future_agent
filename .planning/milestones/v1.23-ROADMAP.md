# Milestone v1.23: Strategy Evidence And Live Resume Readiness

**Status:** SHIPPED 2026-06-21
**Phases:** 48-52
**Total Plans:** 5

## Overview

v1.23 converted the negative forward-paper and backtest evidence into a
read-only strategy readiness workflow. It did not resume live automation,
apply a higher-risk profile, or place orders. The milestone produced the
evidence surface required before a future operator-approved live resume.

## Phases

### Phase 48: Strategy Evidence Baseline

**Goal:** Produce one compact read-only baseline that shows why live resume is
currently blocked and what evidence must improve.

**Requirements:** EVB-01, EVB-02, EVB-03, EVB-04
**Plans:** 1/1 complete
**Status:** Complete - 2026-06-21

**Delivered:**

- Added `ops strategy-evidence-baseline`.
- Combined forward-paper performance, latest outcomes, loss attribution,
  adaptive guard output, server timer state, exchange/manual exposure, and
  confirmation blockers.
- Emitted explicit non-mutation guarantees: no orders, env writes, systemd
  changes, exchange mutations, or `order_intents`.

### Phase 49: Loss-Driven Setup Recalibration

**Goal:** Convert current paper loss attribution into stricter deterministic
setup profiles and point geometry without changing live defaults.

**Requirements:** SRC-01, SRC-02, SRC-03, SRC-04
**Depends on:** Phase 48
**Plans:** 1/1 complete
**Status:** Complete - 2026-06-21

**Delivered:**

- Added profile gates for blocked setup reasons, factor names, required open
  interest, liquidity, momentum, and volume impulse.
- Added `quant_setup_loss_recalibrated` as an explicit backtest/paper variant.
- Kept all live defaults unchanged.

### Phase 50: Multi-Window Hot-Symbol Backtest Matrix

**Goal:** Re-run hot-symbol strategy evidence across multiple recent windows,
intervals, and setup variants to avoid overfitting to one selected interval.

**Requirements:** BFP-01, BFP-02
**Depends on:** Phase 49
**Plans:** 1/1 complete
**Status:** Complete - 2026-06-21

**Delivered:**

- Added `backtest matrix-suite`.
- Added `broad`, `momentum`, and `liquid` hot-universe presets.
- Ran `5m` and `15m` evidence across baseline, guarded, and recalibrated
  variants.
- Phase evidence stayed fail-closed with verdict
  `mixed_candidate_collect_more_data`; `quant_setup_selective_guarded` is only
  a forward-paper candidate, not live-ready.

### Phase 51: Post-Change Forward-Paper Gate

**Goal:** Evaluate new forward-paper evidence separately from older losing
samples so live readiness is based on the current calibrated strategy.

**Requirements:** BFP-03, BFP-04
**Depends on:** Phase 50
**Plans:** 1/1 complete
**Status:** Complete - 2026-06-21

**Delivered:**

- Added `min_profit_factor` to forward-paper performance checks.
- Preserved `since` filtering for post-change evidence slices.
- Kept paper promotion distinct from live resume; even a paper pass does not
  automatically allow live automation.

### Phase 52: Live Resume Readiness Report

**Goal:** Build a single read-only live-resume report that combines strategy
evidence, paper evidence, server state, exchange state, profile state, and
operator confirmation requirements.

**Requirements:** LRR-01, LRR-02, LRR-03, LRR-04
**Depends on:** Phase 51
**Plans:** 1/1 complete
**Status:** Complete - 2026-06-21

**Delivered:**

- Added read-only `ops live-resume-readiness`.
- Combined matrix readiness, strategy baseline, post-change forward-paper
  gates, server timer/service state, signed read-only exposure checks,
  risk-profile preview/readiness, and confirmation requirements.
- Classified manual/unattributed exposure separately from agent-managed
  submitted intents.
- Confirmed the report cannot restore timers, apply `30u_10x_multi_dynamic`,
  or place/cancel/modify Binance orders.

## Milestone Summary

**Key Decisions:**

- Evidence before live resume: negative paper evidence blocks unattended live
  automation.
- Recalibration stays paper-first until repeated matrix and forward-paper
  evidence pass.
- Matrix evidence must cover multiple hot universes and intervals before
  promotion.
- Paper promotion requires PnL, win-rate, profit-factor, and drawdown gates.
- Manual exchange positions, including manual ETH/ETHUSDT, are not agent
  strategy evidence.
- Readiness reporting is not execution authority.

**Issues Resolved:**

- Replaced scattered live-resume interpretation with one read-only readiness
  command.
- Added loss-driven recalibration knobs without changing live defaults.
- Added profit-factor gating to avoid promoting low-quality paper evidence.
- Added manual exposure classification so operator trades are visible but not
  counted as bot-managed positions.

**Issues Deferred:**

- Live automation remains paused until evidence improves and the operator
  separately approves resume.
- Phase 52 readiness command still needs server deployment/read-only execution
  before any live-resume decision.
- Server forward-paper collection still needs enough post-change outcomes for
  the selected guarded variant.

**Technical Debt Incurred:**

- Runtime matrix evidence is stored under `runtime/` by design and must be
  rerun before future decisions.
- `quant_setup_loss_recalibrated` is currently too strict and produced no
  Phase 50 trades, so it needs future tuning or replacement.
