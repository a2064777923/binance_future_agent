# Plan 51-01: Post-Change Forward-Paper Gate

<objective>
Make forward-paper promotion gates evaluate post-change evidence with an
explicit profit-factor threshold while keeping live resume disabled.
</objective>

<tasks>

1. Extend forward-paper performance gate.
   - Preserve existing `since` filtering.
   - Add `min_profit_factor`.
   - Keep all live-resume flags false.

2. Expose CLI threshold.
   - Add `--min-profit-factor` to `ops forward-paper-performance-check`.
   - Pass the threshold through strategy evidence baseline.

3. Add tests.
   - Verify low profit factor blocks paper promotion.
   - Verify strong post-change samples can pass paper promotion while live
     resume remains false.

</tasks>

<success_criteria>

- Operator can evaluate a post-change slice with `--since`.
- Gate requires minimum outcomes, positive PnL, win rate, profit factor, and
  drawdown.
- Live resume remains false even when paper promotion passes.

</success_criteria>
