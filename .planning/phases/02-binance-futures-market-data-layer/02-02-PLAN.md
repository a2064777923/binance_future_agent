<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Candidate symbols -> REST query params | User/config-provided symbols are converted into Binance query params. |
| Binance REST -> metric client | External JSON and status/headers enter local code for metric responses. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Tampering | REST metric params | mitigate | Uppercase and validate symbols, reject blank values and invalid limits, and assert endpoint params in fake-transport tests. |
| T-02-02-02 | Denial of Service | Heavy metric endpoints | mitigate | Require explicit symbols and conservative default limits so the collector cannot accidentally request all symbols across heavy endpoints. |
| T-02-02-03 | Information Disclosure | Metric client | mitigate | Keep unsigned public calls only; do not read Binance API key files or attach credential-bearing headers. |
| T-02-SC | Tampering | Python package dependencies | mitigate | No package-manager installs are authorized; keep the dependency-free standard-library implementation. |
</threat_model>
