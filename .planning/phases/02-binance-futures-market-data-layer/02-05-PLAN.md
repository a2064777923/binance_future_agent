<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment config -> collector | Configured symbols and base URLs drive public market-data collection. |
| CLI args -> collector/writer | User-provided command arguments select symbols and output paths. |
| Collector -> Binance REST client | Public REST responses become normalized local snapshots. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Denial of Service | `MarketDataCollector` | mitigate | Require explicit/configured symbols, reject empty lists, and cap default symbol count before REST calls. |
| T-02-05-02 | Information Disclosure | CLI diagnostics | mitigate | Reuse redaction patterns for config output and keep Binance credentials out of public market-data requests. |
| T-02-05-03 | Tampering | CLI output path | mitigate | Use pathlib/json writer behavior from Plan 02-04, tests with temporary directories, and document caller-provided output as local runtime/data only. |
| T-02-05-04 | Spoofing | fake client injection in tests | accept | Injection is test-only design for D-08; production CLI uses the real public REST client unless an explicit test factory is passed in process. |
| T-02-SC | Tampering | Python package dependencies | mitigate | No new dependency is justified; leave `pyproject.toml` dependencies empty and keep all collector/CLI work standard-library based. |
</threat_model>
