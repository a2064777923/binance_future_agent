<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Binance payload -> normalizer | External REST and WebSocket payload fields become normalized local records. |
| Snapshot writer -> filesystem | Caller-provided paths receive JSONL files under project runtime/data locations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Tampering | REST normalizers | mitigate | Use explicit per-endpoint normalizers with fixture coverage and required metadata assertions. |
| T-02-04-02 | Repudiation | `NormalizedMarketSnapshot` output | mitigate | Include source, event_type, symbol, event_time, and received_at on every snapshot so records are auditable. |
| T-02-04-03 | Information Disclosure | JSONL writer | mitigate | Write public market data snapshots only and avoid config/secrets, account data, user-data streams, AI prompts, or orders. |
| T-02-04-04 | Elevation of Privilege | snapshot path handling | accept | The writer receives local paths from caller code in this developer tool; tests use temporary directories and Phase 5 CLI will constrain default output under runtime/data config. |
| T-02-SC | Tampering | Python package dependencies | mitigate | No JSON/serialization package install is authorized; use Python `json` and `pathlib`. |
</threat_model>
