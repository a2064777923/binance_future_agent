# Plan 22-01: Risk Change Readiness Gate

## Objective

Add a read-only operations gate that decides whether leverage or live risk caps
may be changed.

## Scope

- Add `ops risk-change-check`.
- Reuse live-status exchange evidence so the gate reads positions, normal
  orders, algo orders, and AI backoff without touching exchange state.
- Block risk changes while any active position or open order exists.
- Block risk changes while any submitted order intent lacks a persisted outcome.
- Include an optional `--target-leverage` field in the report for auditability.
- Keep the command observational; no order, cancel, close, or env mutation.

## Verification

- Unit tests cover allowed, active protected position, orphan orders, and
  unreconciled submitted-intent states.
- CLI test covers command routing and fail-closed behavior when exchange
  evidence is missing.
- Server run under the current BNBUSDT live position returns non-zero with
  `active_position_present` and `position_has_algo_protection`.

## Safety Notes

This gate is stricter than `ops resume-check`: timer resume may be safe while a
protected position exists and the one-position cap blocks new entries, but
changing leverage or risk caps should wait until positions/orders are clear and
outcomes are reconciled.
