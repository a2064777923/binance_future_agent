# Plan 43-01: Forward-Paper Loss Attribution And Recalibration

## Goal

Create a read-only attribution report for negative forward-paper performance so
strategy changes can target the losing conditions instead of guessing.

## Scope

- Add `ops forward-paper-loss-attribution`.
- Load paper signals and outcomes for a selected variant, interval, and
  optional start time.
- Join each settled paper outcome back to its originating paper signal.
- Group performance by symbol, side, exit reason, setup reasons, setup
  warnings, and setup factor evidence.
- Rank losing groups by total net PnL, win rate, and sample size.
- Emit recalibration candidates such as symbols to quarantine, sides to
  tighten, exit reasons to inspect, and setup reasons/factors associated with
  losses.
- Keep `live_resume_allowed=false`.

## Out Of Scope

- No live timer restore.
- No risk profile apply.
- No Binance signed endpoint or exchange mutation.
- No automatic filter/profile change from the attribution report.
- No claim that attribution alone proves a profitable strategy.

## Verification

- Focused loss-attribution unit tests.
- CLI smoke for a synthetic paper DB.
- Full local test suite.
- `git diff --check`.
- Secret-pattern scan over the working diff.
- Server deployment and read-only attribution check against the live paper DB.
