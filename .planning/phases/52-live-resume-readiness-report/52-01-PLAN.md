# Plan 52-01: Live Resume Readiness Report

<objective>
Add one read-only command that explains whether live automation can be resumed
by combining matrix evidence, forward-paper evidence, server state, exchange
exposure, risk-profile readiness, and operator confirmation.
</objective>

<tasks>

1. Build readiness report.
   - Reuse strategy evidence baseline for paper, attribution, guard, server,
     and confirmation gates.
   - Add matrix or matrix-suite promotion readiness.
   - Include exposure-status and risk-profile preview without applying changes.

2. Separate manual and agent-managed exposure.
   - Classify active exchange exposure against unreconciled submitted intents.
   - Treat manually supplied symbols as manual exposure.
   - Never count manual exposure as agent strategy evidence.

3. Expose CLI.
   - Add `ops live-resume-readiness`.
   - Keep default report fail-closed until confirmation is removed or supplied
     by a future explicit resume flow.

4. Verify.
   - Cover all-green readiness, manual exposure blocking, matrix-suite mixed
     blocking, and CLI behavior.

</tasks>

<success_criteria>

- One command reports all live-resume blockers in grouped JSON.
- Manual ETH or any other manual position is separated from agent-managed
  submitted intents.
- Live auto-hot is shown as preview-only and remains disabled by default.
- Report has no code path that writes env files, starts timers, applies risk
  profiles, creates order intents, or submits orders.

</success_criteria>
