# Plan 28-01: Dynamic Sizing And Multi-Position Guard

## Objective

Make position size capable of scaling with account state and operator-approved
risk settings, while keeping the 30U live pilot bounded and preserving the
single-position default.

## Scope

- Add config keys for dynamic sizing:
  - `BFA_DYNAMIC_POSITION_SIZING_ENABLED`
  - `BFA_MAX_MARGIN_PER_POSITION_USDT`
  - `BFA_MAX_MARGIN_FRACTION`
  - `BFA_MAX_EFFECTIVE_NOTIONAL_USDT`
  - `BFA_MULTI_POSITION_ENABLED`
- Add a sizing helper that computes a notional cap from:
  - account capital
  - available balance when present
  - leverage
  - max margin per position
  - max margin fraction
  - max risk per trade and stop distance
  - fixed notional fallback
  - exchange minimum executable notional
- Feed the computed cap into AI risk context and final risk validation.
- Keep fixed `BFA_MAX_POSITION_NOTIONAL_USDT` behavior when dynamic sizing is
  disabled.
- Keep multi-position disabled by default.
- When multi-position is enabled, reject same-symbol same-direction duplicate
  exposure and preserve `BFA_MAX_OPEN_POSITIONS`.
- Add tests for 30U/5x conservative sizing, 30U/8x larger sizing, min-notional
  pressure, dynamic disabled fallback, and multi-position guard behavior.

## Verification

- Unit tests for sizing calculations.
- Risk tests prove dynamic caps are enforced before live orders.
- Agent/CLI tests prove AI context sees the computed cap.
- Full local test suite passes.
- Server non-trading test suite passes after deployment.

## Safety Notes

This phase must not edit the server env to raise live leverage, notional, or
open-position caps. It only adds the machinery needed for a later explicitly
approved profile change.
