# Plan 30-01: Portfolio Risk And Multi-Position Profile

## Objective

Let the agent continue scanning and opening controlled additional positions
while an existing position is open, with higher-leverage profiles bounded by
portfolio-level risk budgets.

## Scope

- Add runtime config keys:
  - `BFA_MAX_PORTFOLIO_MARGIN_USDT`
  - `BFA_MAX_PORTFOLIO_MARGIN_FRACTION`
  - `BFA_MAX_PORTFOLIO_NOTIONAL_USDT`
  - `BFA_MAX_SAME_DIRECTION_NOTIONAL_USDT`
- Extend `RiskState` active exposures with notional and margin accounting.
- Add risk gates for:
  - portfolio initial margin after proposed entry;
  - portfolio margin fraction after proposed entry;
  - total portfolio notional after proposed entry;
  - same-direction notional after proposed entry.
- Preserve existing duplicate same-symbol same-direction rejection.
- Let live runner proceed past entry-capacity preflight when
  `BFA_MULTI_POSITION_ENABLED=true` and `BFA_MAX_OPEN_POSITIONS` has remaining
  capacity.
- Evaluate the top-N candidate queue so a same-symbol duplicate or AI pass on
  the first hot coin does not end the whole cycle while other hot symbols remain.
- Let risk-profile readiness evaluate whether protected active positions can be
  carried into the target multi-position profile without breaching portfolio
  caps.
- Add named `30u_10x_multi_dynamic` profile with two positions and portfolio
  caps.
- Extend `ops exposure-status` with portfolio cap context.
- Update examples and README.

## Verification

## Safety Notes

This phase adds machinery and a previewable profile. It does not make the live
bot automatically high leverage. Applying the profile still requires operator
confirmation and the existing apply command.
