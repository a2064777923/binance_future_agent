# Plan 23-01: Closed Outcome Risk Change Strictness

## Objective

Tighten the risk-change readiness gate so only final closed outcomes satisfy the
submitted-trade reconciliation requirement.

## Scope

- Change unreconciled submitted-intent detection to require
  `outcome:{event_id}:closed`.
- Treat `open_or_partial` outcome artifacts as still unreconciled for leverage
  or risk-cap profile changes.
- Add regression coverage for the partial-outcome case.
- Preserve read-only behavior and leave trading/execution paths unchanged.

## Verification

- Unit tests cover that `open_or_partial` outcomes do not clear a submitted
  intent for risk-change readiness.
- Server `ops risk-change-check --target-leverage 8` still blocks while BNBUSDT
  is open and lacks a closed outcome.

## Safety Notes

Open or partial outcome artifacts can be useful for diagnosis, but profile
increases require final realized PnL evidence. This phase prevents a premature
partial artifact from unlocking 8x/10x changes.
