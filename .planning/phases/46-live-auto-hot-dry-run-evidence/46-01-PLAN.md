# Plan 46-01: Live Auto-Hot Dry-Run Evidence

## Goal

Collect server evidence for Phase 45 auto-hot scanning without enabling
unattended live automation.

## Requirements Addressed

- LAD-01
- LAD-02
- LAD-03

## Scope

- Run a one-shot server command with env overrides:
  - `BFA_MODE=dry_run`
  - `BFA_LIVE_AUTO_HOT_SYMBOLS=true`
  - optional conservative `BFA_LIVE_AUTO_HOT_TOP_N`
  - `BFA_AI_FALLBACK_TO_QUANT_ENABLED=true` if needed to avoid AI dependency
- Capture the JSON result fields that prove scanning and bounded evaluation:
  `mode`, `scan_symbols`, `candidate_count`, `evaluated_symbols`, `submitted`,
  `status`, and `risk_reasons`.
- Verify `/etc/binance-futures-agent/env` does not enable live auto-hot after
  the command.
- Verify `binance-futures-agent-paper.timer` is active and
  `binance-futures-agent-live.timer/service` are inactive.
- Record evidence in SUMMARY and VERIFICATION.

## Out Of Scope

- No live timer restore.
- No server env mutation.
- No risk profile, leverage, or sizing change.
- No exchange order, close, or position adjustment.
- No claim that wider scanning is profitable.

## Verification

- Server one-shot dry-run evidence command.
- Server env/service-state check.
- Local `git diff --check`.
- Secret-pattern scan over the working diff.
