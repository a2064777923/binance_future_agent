# Plan 67-01: Adaptive Hot-Symbol Breadth And Guarded Queue

<objective>
Broaden live auto-hot observation to at least 80 Binance USD-M USDT symbols,
add live source-health and per-candidate queue diagnostics, and preserve
weak-evidence, tradability, one-order-per-cycle, and manual-symbol guards.
</objective>

<must_haves>

- D-01: Default live auto-hot scanning must support at least 80 Binance
  USD-M USDT symbols through `BFA_LIVE_AUTO_HOT_TOP_N`.
- D-02: Broader scanning must exclude configured manual symbols before
  market collection, candidate generation, AI decisions, and execution.
- D-03: Broader observation is not a risk increase and must keep existing
  deterministic gates plus the one-order-per-cycle default.
- D-04: Source health must include Binance ticker, klines, open interest,
  open-interest history, funding, taker buy/sell flow, manual Square exports,
  RSS feeds, and market-heat fallback where available.
- D-05: Source health must report counts/status/configured inputs without
  exposing secrets and must treat missing optional narrative sources as
  non-fatal.
- D-06: Source-health shape should stay consistent with forward-paper
  source-health output where practical.
- D-07: Forward-paper guard behavior remains risk-reducing only and must
  not raise size, leverage, or concurrency.
- D-08: Guard diagnostics must distinguish symbol, side, and factor blocks
  and include enough summary counts to show active versus insufficient evidence.
- D-09: Guard reset/override and promotion remain evidence-gated; Phase 67
  must not add automatic risk increases.
- D-10: The runner must continue after retryable setup, AI, exchange-filter,
  guarded suppression, and candidate-local symbol/risk skips.
- D-11: The runner must stop after a submitted order or a non-retryable
  rejection, preserving at most one live order per cycle.
- D-12: The final result must include both final selected symbol and full
  evaluated-symbol/per-candidate path for Phase 66 explainability.
- D-13: `BTWUSDT` and other manual symbols remain operator-owned and
  excluded from bot capacity, auto-management, close/reduce execution, and
  candidate selection.
- D-14: Runtime caps must be read from config; do not hard-code the current
  manually widened server values into local code.

</must_haves>

<tasks>

1. Raise the live auto-hot breadth default.
   - Read first: `src/bfa/config.py`, `.env.example`,
     `deploy/server-env.example`, and README live automation docs.
   - Change the default `BFA_LIVE_AUTO_HOT_TOP_N` from `40` to `80`.
   - Update env examples and README wording to describe the broader but still
     configurable auto-hot universe.
   - Do not change server env or systemd units in this phase.

2. Add live source-health reporting.
   - Read first: `src/bfa/agent.py`, `src/bfa/cli.py`
     `_forward_paper_source_health()`, `src/bfa/ops/forward_paper.py`,
     `src/bfa/market/collector.py`, and `src/bfa/strategy/features.py`.
   - Extend `AgentRunResult` with `source_health`.
   - Refactor live symbol scanning so `_agent_scan_symbols()` behavior is
     preserved but a new internal helper can also return source-health details.
   - Include ticker selection status, ticker payload count, selected rows,
     filters, selected/excluded manual symbols, fallback mode, and selected
     symbol count.
   - Summarize collected market snapshots by event type and covered symbol,
     including kline/ticker/funding/open-interest/open-interest-history/taker
     flow when present.
   - Summarize narrative collection by source, configured Square export
     directory, RSS feed count, market-heat fallback status/count, and covered
     symbols.
   - Include a compact paper-guard status summary and guarded symbols/factors.
   - Make sure all no-candidate, ai-pass, quant-pass, rejected, and submitted
     return paths include the same source-health field.

3. Add per-candidate queue diagnostics.
   - Extend `AgentRunResult` with `candidate_evaluations` or an equivalent
     stable list.
   - For each evaluated candidate, record symbol, candidate score, setup
     decision/reasons, AI status/validation status, execution status, risk
     reasons, and whether the runner continued to the next candidate.
   - Keep `selected_symbol` semantics as the final evaluated/submitted symbol
     and keep `evaluated_symbols` for compatibility.
   - Do not persist a new table unless tests show existing event artifacts are
     insufficient; prefer result payload diagnostics for this phase.

4. Tighten retryable queue behavior without relaxing global risk gates.
   - Review `_should_try_next_candidate()` and the setup/AI/risk branches in
     `_evaluate_candidate_queue()`.
   - Ensure setup pass, AI pass, duplicate same-symbol-direction exposure,
     missing symbol filters, min-quantity/min-notional filter failures, guarded
     factor suppression, and candidate-local notional/risk cap failures can
     move to later candidates.
   - Keep global/account blockers non-retryable: daily loss cap, cooldown,
     kill switch, missing credentials, max open positions, multi-position
     disabled, portfolio margin cap, portfolio notional cap, and same-direction
     cap.
   - Preserve the default one-order-per-cycle behavior: return immediately once
     an order is submitted or a non-retryable rejection occurs.

5. Preserve manual-symbol boundaries.
   - Verify `BFA_MANUAL_POSITION_SYMBOLS` is applied before auto-hot selected
     symbols are used for market collection, narrative known symbols, candidate
     generation, AI, and execution.
   - Add/extend tests showing `BTWUSDT` is excluded from a broad 80-symbol
     auto-hot list while remaining visible through lifecycle/manual diagnostics
     from prior phases.
   - Do not add a manual-to-bot transfer path.

6. Add focused tests.
   - Extend `tests/test_agent_runner.py` to cover:
     - 80-symbol auto-hot selection.
     - source-health payload for ticker, market snapshots, narrative sources,
       market-heat fallback, manual exclusions, and paper guard.
     - candidate evaluation path for setup pass, AI pass, retryable risk skip,
       and eventual submitted/dry-run candidate.
     - non-retryable account/global blockers stop the queue.
     - manual `BTWUSDT` excluded from broad scanning.
   - Extend `tests/test_config.py` for the new default.
   - Extend `tests/test_deploy_assets.py` for env example defaults.
   - Keep fixtures synthetic and secret-free.

7. Update documentation.
   - Update README live automation text to mention the default 80-symbol
     auto-hot universe, source-health output, candidate queue diagnostics, and
     unchanged one-order-per-cycle/manual-symbol guard behavior.
   - Avoid claiming profitability or Lana-equivalent performance.

8. Run local verification.
   - Run focused tests:
     `python -m unittest tests.test_agent_runner tests.test_config tests.test_deploy_assets`.
   - Run full tests: `python -m unittest discover -s tests`.
   - Run `git diff --check`.
   - Run `node $HOME/.codex/gsd-core/bin/gsd-tools.cjs query audit-open --json`.

9. Close GSD phase locally.

     verification evidence.
   - Write `67-VERIFICATION.md` mapping SCAN-01 through SCAN-04 to current
     evidence.
   - Update `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`,
     `.planning/STATE.md`, and `.planning/PROJECT.md` through GSD-safe
     handlers or narrow planning edits where no handler exists.
   - Commit implementation and closeout artifacts.

</tasks>
