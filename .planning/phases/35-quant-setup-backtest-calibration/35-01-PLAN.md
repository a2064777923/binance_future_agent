# Plan 35-01: Quant Setup Backtest Calibration

## Goal

Backtest the same deterministic setup logic used by the live runner instead of
only the older fixed hot-momentum strategy.

## Scope

- Add a `quant_setup` backtest variant that uses `bfa.strategy.setup` to
  generate side, stop, target, notional, and hold-time from completed kline
  windows.
- Preserve existing `strict`, `balanced`, and `aggressive` hot-momentum
  variants.
- Support both long and short setup-driven trades in the backtest engine.
- Allow CLI `backtest run`, `backtest sweep`, and matrix reporting to accept
  `quant_setup`.
- Keep live behavior unchanged; this phase is offline calibration only.

## Verification

- Unit tests for setup-driven long and short backtest trades.
- Matrix tests proving `quant_setup` is accepted as a variant.
- CLI smoke tests for `backtest run --variant quant_setup`.
- Full local test suite.
- CLI manual smoke for `backtest run` and `backtest sweep` with generated
  kline data.
