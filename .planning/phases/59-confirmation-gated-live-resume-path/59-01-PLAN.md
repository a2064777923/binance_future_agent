# Plan 59-01: Confirmation-Gated Live Resume Path

<objective>
Add a dedicated live-resume action-plan preview and confirmed apply path that
can describe env/profile/timer changes without mutation, and that refuses to
mutate unless the operator decision packet is `eligible_for_operator_resume`
and the action confirmation token matches.
</objective>

<tasks>

1. Add read-only live resume action-plan builder.
   - Create `src/bfa/ops/live_resume_plan.py`.
   - Accept a current `AppConfig`, an operator decision packet or packet
     artifact, target profile name, and current/target systemd states.
   - Include target risk profile values and diff from
     `build_risk_profile_plan`.
   - Include planned live timer/service and paper timer/service changes.
   - Include readiness artifact path/status when available.
   - Emit a confirmation token derived from the packet status, target profile,
     target profile values, and systemd change plan.
   - Emit explicit read-only proof:
     `writes_env_files=false`, `changes_systemd_state=false`,
     `mutates_exchange_state=false`, `places_orders=false`,
     `creates_order_intents=false`.

2. Add confirmed apply wrapper.
   - Require exact confirmation token match.
   - Require operator packet status `eligible_for_operator_resume`.
   - Require live service inactive before any mutation.
   - Reuse `apply_risk_profile` for env writes, including backup behavior.
   - Apply only injectable/systemd command-list operations in code; tests use a
     fake applier so no unit test mutates host systemd.
   - Return a structured report with `applied`, `reasons`,
     `risk_profile_apply`, and `systemd_actions`.

3. Add CLI routes.
   - Add `ops live-resume-plan` for preview.
   - Add `ops live-resume-apply` for confirmed apply.
   - Support `--operator-decision-report` for reading an existing packet JSON.
   - Support `--target-profile`, timer target flags, service-state overrides,
     env path, db path, and `--confirm-token`.
   - Keep preview exit code zero when the report is generated, even if it says
     resume is not allowed; apply exits non-zero unless mutation actually
     applies.

4. Add focused tests and docs artifacts.
   - Unit-test preview for non-mutation proof and 30U/10x bounded caps.
   - Unit-test blocked apply for `collect_more_paper`, missing token, token
     mismatch, and active live service.
   - Unit-test eligible confirmed apply with fake risk-profile/systemd appliers.
   - CLI-test preview from a saved operator decision artifact and blocked apply
     from a non-eligible packet.
   - Write Phase 59 summary and verification artifacts after implementation.

</tasks>

<success_criteria>

- Operator can preview exact target profile, systemd changes, readiness/packet
  status, confirmation token, and non-mutation proof.
- Confirmed apply refuses to write env or change systemd unless the packet is
  `eligible_for_operator_resume` and the confirmation token matches.
- The target `30u_10x_multi_dynamic` path exposes and preserves account,
  notional, margin, concentration, per-trade risk, and daily-loss caps.
- No Phase 59 path places orders, cancels orders, creates order intents, or
  mutates Binance exchange state.
</success_criteria>
