# Plan 34-01: Deterministic Quant Setup And Trade Trace

## Goal

Move entry, stop, target, hold-time, and notional generation out of the AI
response and into a deterministic, auditable multi-factor setup layer.

## Scope

- Extend strategy feature extraction to preserve recent kline-window and flow
  signals instead of only the latest single-point values.
- Add a deterministic multi-factor trade setup engine that scores momentum,
  liquidity, open interest, taker flow, funding, volatility, narrative quality,
  and pilot tradability.
- Make AI a setup overlay/veto: accepted AI trade responses must echo the
  deterministic setup prices, notional, and hold time.
- Add optional `BFA_AI_FALLBACK_TO_QUANT_ENABLED` behavior so an explicitly
  configured deployment can use the deterministic setup when AI is unavailable.
- Persist trade setups for new agent cycles and expose read-only `ops
  trade-trace` to reconstruct candidate, setup, AI, risk, intent, and exchange
  evidence.
- Preserve live safety: do not restore the live timer, execute adjustment
  orders, apply risk profiles, or mutate exchange state as part of this phase.

## Verification

- Unit tests for deterministic long/short/pass setup behavior.
- AI schema and validation tests proving setup context is compact and AI cannot
  modify setup prices.
- Agent and CLI tests proving setup persistence, AI overlay, fallback plumbing,
  and trade trace reconstruction.
- Full local test suite.
- Server deployment with full server test suite.
- Read-only server `ops trade-trace --symbol SOLUSDT` against the existing
  live order evidence.
