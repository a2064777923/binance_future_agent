# Plan 48-01: Strategy Evidence Baseline

<objective>
Add one compact, read-only strategy evidence baseline command that explains
why live automation remains blocked and what evidence is driving that verdict.
</objective>

<tasks>

1. Add `bfa.ops.strategy_evidence_baseline`.
   - Combine forward-paper performance, loss attribution, and adaptive guard
     evidence.
   - Include read-only server automation state for `paper.timer`,
     `live.timer`, and `live.service`.
   - Group live-resume blockers into `strategy_evidence`, `server_state`,
     `exchange_state`, and `confirmation`.

2. Add CLI entrypoint.
   - Expose `python -m bfa.cli ops strategy-evidence-baseline`.
   - Accept the same paper evidence filters as existing paper commands.
   - Allow server state overrides while defaulting to read-only systemd checks.

3. Add tests.
   - Verify negative paper performance blocks live resume with compact metrics.
   - Verify server/exchange/confirmation blockers are grouped.
   - Verify the CLI returns JSON and nonzero when live resume is blocked.

</tasks>
