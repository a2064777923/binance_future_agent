# Plan 11-01: Reference-Price AI Decision Robustness

## Objective

Reduce live-cycle waste from AI outputs that say `trade` but omit executable
entry, stop, or target prices, without weakening deterministic risk gates.

## Scope

- Extract latest kline close into strategy features as `reference_price`.
- Include `reference_price` in compact AI decision context.
- Tighten decision instructions:
  - if `decision=trade`, entry, stop, target, notional, hold time, and side must
    be complete;
  - if a complete trade cannot be formed from the candidate and risk limits, the
    model must return `pass`;
  - entry should stay close to `reference_price`, with stop/target geometry that
    passes risk validation.
- Add deterministic validation for trade entry distance from reference price.
- Add tests proving incomplete trade payloads fail closed and reference-price
  context reaches the AI client.

## Verification

- Unit tests cover feature extraction, compact AI context, decision validation,
  and agent runner AI context payload.
- Full unit suite passes.
- `git diff --check` passes.
- Server deploy health check passes after implementation.

## Safety Notes

This phase must not raise live risk caps, change Binance credentials, or bypass
protective-order requirements. It only improves decision quality and fail-closed
classification.
