# Plan 50-01: Multi-Window Hot-Symbol Backtest Matrix

<objective>
Make a refreshed matrix run cover multiple hot-symbol universes, intervals, and
setup variants in one reproducible report.
</objective>

<tasks>

1. Add matrix-suite support.
   - Define reusable hot-universe presets.
   - Run existing matrix logic once per preset.
   - Aggregate variant evidence across presets.

2. Expose CLI.
   - Add `python -m bfa.cli backtest matrix-suite`.
   - Default to `5m,15m` and baseline/recalibrated quant setup variants.
   - Support output to a JSON report.

3. Generate Phase 50 evidence.
   - Run the suite against public Binance market data.
   - Store the report under `runtime/`.

4. Verify.
   - Add focused tests for suite payload and CLI output.
   - Run full local verification before commit.

</tasks>

<success_criteria>

- One command/report covers multiple hot universes, multiple intervals, and
  multiple setup variants.
- Report includes trade count, total net PnL, win rate through sweep summaries,
  positive-window rate, profit factor through window summaries, and worst
  drawdown.
- Promotion remains fail-closed when evidence is negative or thin.
- Full local tests pass.

</success_criteria>
