# Plan 61-01: Close-Review Exit Plan Repair

<objective>
Add an operator-readable, filter-aware lifecycle diagnostic to
`ops position-adjustment-plan` so every active position explains why it is
`manual_hold`, `hold`, `watch`, `reduce`, `close_ready`, or `blocked`, while
preserving the existing confirmation-gated execution path.
</objective>

<tasks>

1. Add position lifecycle diagnostics.
   - Create an additive diagnostic dataclass in
     `src/bfa/ops/position_adjustment.py`.
   - Include per-position fields for symbol, position side, source
     recommendation, lifecycle decision, urgency, manual flag, protection
     state, matching-intent state, exchange-filter state, failed preconditions,
     passed preconditions, and optional order plan.
   - Populate diagnostics from `PositionReviewReport.positions` and existing
     adjustment plan items.

2. Preserve execution semantics.
   - Keep `plans` as the actionable/blocked candidate list used by
     confirmation-gated execution.
   - Do not add manual positions to `plans`.
   - Ensure manual diagnostics do not make an otherwise ready agent-managed
     close/reduce plan report blocked.

3. Clarify filter-aware close readiness.
   - For a filter-passing `close_review` position, emit lifecycle decision
     `close_ready` and include the `full_close` order plan.
   - For a filter-failing `close_review` position, emit lifecycle decision
     `blocked` and expose exact failed preconditions such as
     `symbol_filters_missing`, `quantity_below_min`, or
     `full_close_quantity_not_step_aligned`.

4. Prove manual and urgency boundaries.
   - Add tests showing manual `BTWUSDT` remains `manual_hold`, has no order
     plan, and does not block an agent-managed close plan.
   - Add tests showing unprotected or deteriorating positions keep higher
     urgency than normal hold-time expiry.

5. Verify and deploy.
   - Run focused tests for position adjustment and CLI output.
   - Run the full local test suite.
   - Deploy to the isolated server only after local tests pass.
   - Run server focused/full tests and a read-only server
     `position-adjustment-plan` smoke artifact.

6. Close GSD phase.
   - Write summary and verification artifacts.
   - Update ROADMAP, REQUIREMENTS, STATE, and PROJECT.
   - Commit the Phase 61 implementation and closeout.

</tasks>
