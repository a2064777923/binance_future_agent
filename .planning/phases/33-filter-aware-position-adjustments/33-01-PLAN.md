# Plan 33-01: Filter-Aware Position Adjustments

## Goal

Ensure active-position adjustment plans only expose reduce orders that can
survive Binance symbol filters.

## Scope

- Load exchangeInfo for adjustment planning.
- Round partial take-profit quantities down to `stepSize`.
- Block partial take-profit plans that would fail `minQty` or `minNotional`.
- Require exchange filters before confirmed adjustment execution.
- Preserve read-only preview behavior and confirmation gates.

## Verification

- Focused tests for adjustment planning, CLI, and agent runner.
- Full local test suite.
- Server deploy and read-only preview.
