# Plan 13-01: Pilot Symbol Universe

## Objective

Replace the original BTC/ETH/SOL default market-symbol list with a controlled
10-symbol Binance USD-M universe that currently fits the 20 USDT
max-position-notional pilot cap.

## Scope

- Use current Binance public `exchangeInfo`, ticker price, and 24h ticker data
  to identify high-liquidity USDT perpetual symbols whose minimum executable
  notional is at or below 20 USDT.
- Choose at most 10 symbols so the existing `MarketDataCollector.max_symbols`
  cap remains valid.
- Update `BFA_MARKET_SYMBOLS` defaults in code and env templates.
- Keep all risk caps unchanged:
  - account capital 100 USDT;
  - max leverage 3x;
  - max position notional 20 USDT;
  - max risk per trade 1 USDT;
  - max daily loss 3 USDT.
- Update tests that depended on old BTC/ETH fixture defaults to pass an explicit
  fixture allowlist.

## Selected Universe

`HYPEUSDT,SOLUSDT,ZECUSDT,WLDUSDT,XRPUSDT,AVAXUSDT,BNBUSDT,DOGEUSDT,NEARUSDT,ADAUSDT`

## Verification

- Config tests prove defaults parse to the selected 10 symbols.
- CLI fixture tests explicitly override their fixture allowlist.
- Full unit suite passes.
- `git diff --check` passes except Windows LF-to-CRLF warnings.
- Server env is updated out of band after deployment.

## Safety Notes

This phase expands opportunity surface but not risk size. It deliberately avoids
raising caps to make BTC/ETH fit. The existing pilot tradability filter remains
active if future Binance filters or prices make any selected symbol incompatible.
