# Plan 37-01: Strategy Promotion Gate

## Goal

Turn recent matrix backtest evidence into an explicit read-only gate so a
strategy cannot be treated as ready for live resume or higher-risk operation
just because code and tests pass.

## Scope

- Add a strategy promotion check that reads a `bfa_hot_backtest_matrix_v1`
  report.
- Require the selected variant summary to be promoted, positive, and below
  drawdown cap.
- Require every interval cell to pass verdict, trade count, positive PnL,
  positive-window-rate, and drawdown checks.
- Add a CLI command under `ops`.
- Record the negative Phase 36 `quant_setup` matrix result as a
  keep-live-paused condition.

## Out Of Scope

- No live order or exchange mutation.
- No risk-profile apply, live timer restore, or server env change.
- No attempt to optimize parameters inside this phase.

## Verification

- Unit tests for pass, fail, and missing report behavior.
- CLI test for a negative matrix.
- Manual check against `runtime/quant_setup_matrix_phase36.json`.
- Full local test suite.
