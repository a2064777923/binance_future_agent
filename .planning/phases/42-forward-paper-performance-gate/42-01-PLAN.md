# Plan 42-01: Forward-Paper Performance Gate

## Goal

Turn accumulated forward-paper signals and outcomes into a read-only
performance gate that can decide whether paper evidence is promising, while
still refusing to resume live automation from paper evidence alone.

## Scope

- Add `ops forward-paper-performance-check`.
- Read `paper_signals` and `paper_outcomes` from the event store for a selected
  variant, interval, and optional start time.
- Summarize signal count, settled outcome count, open signal count, win/loss
  counts, win rate, net PnL, average PnL, profit factor, worst drawdown, exit
  reasons, symbol breakdowns, and latest outcomes.
- Compare evidence against minimum outcomes, minimum win rate, minimum net PnL,
  and maximum drawdown thresholds.
- Return explicit statuses for missing evidence, insufficient outcomes,
  failing evidence, and promising paper evidence.
- Keep `live_resume_allowed=false` for every status.
- Add focused unit and CLI tests.

## Out Of Scope

- No live timer restore.
- No risk profile apply.
- No signed Binance endpoint or exchange mutation.
- No claim that paper evidence proves profitability.
- No automatic conversion from paper promotion to live execution.

## Verification

- Focused forward-paper performance tests.
- CLI smoke for an empty database.
- Full local test suite.
- `git diff --check`.
- Secret-pattern scan over the working diff.
- Server deployment and read-only performance check against the live paper DB.
