# Plan 18-01: DeepSeek Provider Switch

## Objective

Switch live AI decisions from the intermittent OpenAI-compatible endpoint to
DeepSeek while preserving deterministic validation, backoff behavior, and all
100 USDT pilot risk caps.

## Triggering Evidence

The server live timer reached the AI decision layer after Phase 17, but the
previous OpenAI-compatible endpoint intermittently returned invalid JSON or
timed out under the configured fail-closed timeout. The user approved switching
to DeepSeek and provided the API key out of band.

## Scope

- Add runtime provider selection with `BFA_AI_PROVIDER=openai|deepseek`.
- Keep OpenAI Responses API support as the default/fallback provider.
- Add a DeepSeek Chat Completions client using JSON object mode.
- Reuse the existing compact decision context, deterministic schema checks,
  local risk validation, AI backoff, and redacted journaling.
- Let fenced or prefixed JSON be extracted before strict schema validation so
  model-format noise rejects less often without relaxing required fields.
- Update local/server env examples and deployment docs without committing real
  secret values.
- Deploy to the isolated server path and switch only the selected AI provider
  values in `/etc/binance-futures-agent/env`.

## Verification

- Unit tests cover provider selection, DeepSeek request payload shape, JSON
  extraction, config validation, health checks, and deploy asset hygiene.
- Full unit suite passes locally before deployment.
- Local DeepSeek smoke test returns parseable structured output using a key held
  only in the process environment.
- Server focused tests and health checks pass after deployment.
- One live service/timer cycle runs without service crash and without changing
  Binance credentials, risk caps, margin mode, position mode, or systemd
  isolation.

## Safety Notes

The LLM remains a slow-path analyst/veto layer. If the selected AI provider is
slow, unavailable, or returns invalid output, the runner records AI backoff or
rejection and stops before execution. This phase does not increase leverage,
position size, daily loss, symbol universe, or order authority.
