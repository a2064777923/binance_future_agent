# Plan 27-01: Operator-Approved Time Exit Execution

## Objective

Add a live-capable time-exit execution command that can close an overdue,
protected position only after an operator confirms the latest generated plan.

## Scope

- Add `ops time-exit-execute`.
- Re-run signed Binance evidence collection and Phase 26 time-exit planning
  immediately before any execution attempt.
- Require live mode and fail closed when the live service is active.
- Default to `confirmation_required` and emit a plan-derived confirmation token.
- Execute only when `--confirm-token` exactly matches the current plan token.
- Submit the planned market close order.
- Cancel remaining open algo orders for the same symbol after the close order
  is submitted.
- Persist close response, algo cleanup response or error, confirmation token,
  and source plan evidence as a `time_exit` exchange response.
- Add unit, client, and CLI tests.

## Verification

- Unit tests prove no order is placed without the token.
- Unit tests prove a matching token submits the close order and cancels algo
  orders through fake clients.
- CLI tests prove the command returns `confirmation_required` without a token.
- Signed client tests prove the algo cleanup endpoint is correctly signed.
- Full local and server test suites pass.
- Server verification must not execute the live close order unless the operator
  explicitly approves it.

## Safety Notes

This phase adds the missing manual intervention tool but keeps it fail-closed.
Deploying the command is not approval to run it against BNBUSDT or any future
position. Live execution requires a fresh token from the latest server response
and explicit operator approval.
