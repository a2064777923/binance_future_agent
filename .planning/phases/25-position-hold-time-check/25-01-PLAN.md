# Plan 25-01: Position Hold-Time Check

## Objective

Add a read-only operator check that reports whether active live positions have
exceeded the AI decision's suggested `hold_time_minutes` while preserving all
existing protective-order and risk-change gates.

## Scope

- Add `ops position-hold-check`.
- Reuse `live-status` exchange evidence rather than adding a new Binance read
  path.
- Match active positions to unclosed submitted order intents by symbol and
  position side.
- Extract `metadata.hold_time_minutes` from the submitted intent.
- Report elapsed minutes, overdue status, unrealized PnL, protection count,
  and reasons.
- Return non-zero when an active position requires review, lacks protection, or
  exchange evidence is missing.
- Keep the command read-only: no closes, cancels, leverage changes, env edits,
  or timer changes.
- Add unit and CLI tests.

## Verification

- Unit tests cover within-window, expired-window, missing-protection, and
  missing-exchange-evidence states.
- CLI tests cover fake signed-client injection and expired-hold exit behavior.
- Full local test suite passes.
- Server focused/full tests pass after deployment.
- Server live `ops position-hold-check` reports the current BNBUSDT position
  without mutating exchange state.

## Safety Notes

This phase does not implement automatic time-based exits. It only makes the
current position's time-overrun state explicit so a later phase can decide
whether to add operator-approved or risk-gated automated time exits.
