# Plan 58-01: Promotion Matrix And Loss Review

<objective>
Make Phase 58 evidence review explicit: promotion checks must identify whether
the strategy should collect more paper, move to forward-paper candidacy, or is
only strategy-eligible for a later live-resume gate; manual loss incidents must
be compared against deterministic risk and paper-guard rules.
</objective>

<tasks>

1. Extend strategy promotion output.
   - Preserve existing `status`/`promotion_allowed` behavior where practical.
   - Add an explicit `promotion_stage` field with values such as
     `collect_more_paper`, `forward_paper_allowed`, and `live_resume_eligible`.
   - Add an evidence-boundary field that states public Lana/Square/X claims are
     design inputs only and not promotion evidence.

2. Add read-only manual loss review.
   - Load `manual_loss_incident` artifacts from `risk_state`.
   - Compare each incident with current config/risk guards:
     max leverage, stop-loss status, liquidation distance, and current
     forward-paper symbol/side blocks.
   - Output per-incident `guard_outcome`, triggered rules, warnings, and a
     summary count.
   - Keep the command read-only: no env writes, no systemd changes, no signed
     exchange mutations, and no orders.

3. Add CLI route and tests.
   - Expose `python -m bfa.cli ops manual-loss-review`.
   - Unit-test no incidents, risk-blocked incidents, paper-guard symbol/side
     comparison, and not-caught incidents.
   - CLI-test the new route and updated promotion fields.

4. Capture Phase 58 evidence artifacts.
   - Write summary and verification files.
   - Update roadmap/state/requirements status only after verification.

</tasks>

<verification>

- `python -m unittest tests.test_ops_manual_loss_review tests.test_ops_strategy_promotion tests.test_cli`
- `python -m unittest discover -s tests`
- `git diff --check`

  recorded manual loss.
- Optional server read-only smoke after deploy.

</verification>

<success_criteria>

- Promotion output clearly distinguishes collect-more-paper, forward-paper, and
  later live-resume eligibility states.
- Manual loss incidents can be reviewed against deterministic risk/guard rules
  without mutation.
- Public claims are explicitly excluded as promotion evidence.
- The phase remains read-only with respect to Binance, env files, timers,
  services, and open positions.
</success_criteria>
