# Plan 45-01: Live Auto-Hot Candidate Breadth

## Goal

Let the live/dry-run agent optionally scan a wider auto-hot symbol universe
without changing actual order authority.

## Requirements Addressed

- LAC-01
- LAC-02
- LAC-03
- LAC-04

## Scope

- Add config defaults and validation for live auto-hot scanning:
  - enable flag;
  - top-N;
  - minimum quote volume;
  - minimum absolute 24h price change percent.
- Reuse the existing `select_hot_usdt_symbols` ranking logic.
- Add an agent helper that resolves the per-cycle scan symbols:
  auto-hot symbols when enabled and non-empty; otherwise `BFA_MARKET_SYMBOLS`.
- Use the resolved scan symbols for:
  - market data collector;
  - manual/RSS narrative known symbols;
  - market-heat fallback;
  - replay packet `symbols`;
  - `StrategyConfig.allowed_symbols`.
- Keep `run_agent_once(top_n=...)` semantics unchanged: top-N controls how many
  ranked candidates can be evaluated, not how many symbols are scanned.
- Update env examples and deployment docs to document the new variables with
  live auto-hot disabled by default.
- Add tests for config defaults, fallback behavior, auto-hot selection, and
  unchanged candidate queue/order authority.

## Out Of Scope

- No live timer restore.
- No server env mutation.
- No leverage, sizing, max open position, or risk-profile change.
- No exchange order, close, or position adjustment.
- No claim that wider scanning is profitable.

## Verification

- Focused config and agent-runner tests.
- Deploy asset tests for secret-safe env examples.
- Full local test suite.
- `git diff --check`.
- Secret-pattern scan over the working diff.

## Done

- A dry-run/live cycle can be configured to scan more than the fixed 10-symbol
  list, but defaults remain conservative.
- Paper-only behavior remains separate.
- The implementation is documented and verified without touching live server
  state.
