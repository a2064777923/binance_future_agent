# Plan 16-01: Position Mode And Entry Fail-Closed

## Objective

Match the live Binance account's position-side mode explicitly and make entry
order rejections fail closed instead of crashing the systemd service.

## Triggering Evidence

After deploying cross margin mode, the live runner reached Binance entry order
placement for HYPEUSDT. Binance rejected the order with "Order's position side
does not match user's setting." No order was submitted, but the service crashed
because entry order errors were not yet caught.

## Scope

- Add `BFA_POSITION_MODE` config with accepted values:
  - `one_way`;
  - `hedge`.
- Keep default `BFA_POSITION_MODE=one_way` in code and templates.
- In hedge mode, send:
  - `positionSide=LONG` for long/BUY entries;
  - `positionSide=SHORT` for short/SELL entries.
- Include the same `positionSide` on protective close-position algo orders.
- Catch Binance entry-order errors and persist rejected evidence with
  `entry_order_failed`.
- Keep all pilot risk caps, margin mode, kill switch, and protective-order
  requirements unchanged.

## Verification

- Config tests cover invalid position mode and live hedge warning.
- Signed client tests cover `positionSide` serialization.
- Execution tests cover hedge position side on entry/protective orders.
- Execution tests cover fail-closed entry order rejection.
- Full unit suite passes.
- Server env is updated explicitly to `BFA_POSITION_MODE=hedge`.
- Server health check and live timer observation pass after deployment.

## Safety Notes

Hedge mode changes only Binance order metadata needed to match the account
setting. It does not increase notional, leverage, per-trade risk, daily loss, or
position count caps.
