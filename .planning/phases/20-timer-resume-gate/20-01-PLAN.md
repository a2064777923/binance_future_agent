# Plan 20-01: Timer Resume Gate

## Objective

Add a read-only operational gate that decides whether the live timer can be
resumed after a paused live-position review.

## Scope

- Add `ops resume-check`.
- Reuse live-status exchange evidence rather than introducing new write paths.
- Return `resume_allowed` only when positions, normal open orders, algo open
  orders, and AI backoff are all clear.
- Return `keep_paused` for an active position with confirmed algo protection.
- Return `urgent_attention` for active positions without confirmed protection or
  open orders without an active position.
- Do not automatically start, stop, cancel, close, or modify any Binance order
  or systemd unit.

## Verification

- Unit tests cover clear exchange state, protected active position, unprotected
  active position, orphan orders, and AI backoff.
- CLI test covers `ops resume-check` JSON output and exit code.
- Full unit suite passes.
- Server deployment of the command is verified read-only against the current
  ZECUSDT position; expected result is `keep_paused` and non-zero exit.
- Server `binance-futures-agent-live.timer` remains inactive after verification.

## Safety Notes

This phase deliberately prevents a false green resume decision. Any live
position or inconsistent exchange state keeps automation paused until explicitly
reviewed.
