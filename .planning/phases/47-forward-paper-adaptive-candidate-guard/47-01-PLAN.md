# Plan 47-01: Adaptive Forward-Paper Guard

## Objective

Convert repeated negative forward-paper outcomes into a deterministic,
read-only guard that can quarantine recent losing symbols and reject or
penalize losing sides/factor conditions before the agent or paper recorder
keeps sampling them.

## Tasks

1. Add guard configuration defaults and validation.
   - `BFA_FORWARD_PAPER_GUARD_ENABLED`
   - `BFA_FORWARD_PAPER_GUARD_VARIANT`
   - `BFA_FORWARD_PAPER_GUARD_INTERVAL`
   - minimum outcome thresholds
   - symbol/side/factor PnL and win-rate thresholds
   - score penalty control

2. Add a read-only guard module.
   - Load joined paper signal/outcome evidence from SQLite.
   - Summarize symbol, side, and factor-reason groups.
   - Emit `status`, `reasons`, `summary`, `symbol_blocks`,
     `side_blocks`, and `factor_penalties`.
   - Return no-op when evidence is insufficient.

3. Wire the guard into agent candidate generation and setup evaluation.
   - Reject blocked symbols in `generate_candidates`.
   - Penalize candidates carrying blocked/weak factor evidence.
   - Pass guard blocks into `build_trade_setup` via profile data.
   - Include guard reason codes in candidate rejection and setup pass reasons.

4. Wire the guard into forward-paper runs.
   - Build the same guard before new paper signal generation.
   - Skip blocked symbols before building paper signals.
   - Persist no order intents and mutate no exchange state.
   - Report guard status and guard skip counts.

5. Add focused tests.
   - Guard no-ops with insufficient evidence.
   - Guard blocks repeated losing symbols.
   - Agent rejects blocked symbols before AI.
   - Forward-paper reports guard skips.
   - Config/deploy examples include safe defaults.

6. Verify locally and, if green, deploy code-only to the server with paper timer
   paused/restored and live service/timer left inactive.

## Acceptance Criteria

- Guard decisions are deterministic from local paper evidence.
- Without enough outcomes, behavior remains unchanged.
- With repeated losing symbol evidence, agent candidates reject that symbol.
- Forward-paper skips blocked symbols and reports why.
- The guard never writes `order_intents`, env files, timers, or exchange orders.
- Full local and server tests pass before any operational claim.
