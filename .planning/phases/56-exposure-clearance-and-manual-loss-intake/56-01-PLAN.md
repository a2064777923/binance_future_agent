# Plan 56-01: Exposure Clearance And Manual Loss Intake

<objective>
Add the shortest safe live-resume precursor: a read-only exposure clearance
packet that classifies active exchange/local evidence and an append-only manual
loss incident intake command. The phase must make the current
`resolve_exposure` state actionable without changing live env, timers,
services, Binance positions, or orders.
</objective>

<tasks>

1. Add exposure clearance report module.
   - Create `src/bfa/ops/exposure_clearance.py`.
   - Reuse `build_exposure_status_report` for signed read-only Binance/local
     evidence.
   - Classify each active position as `agent_managed`, `manual`,
     `stale_attributed`, or `unknown` from active exchange exposure, manual
     symbol overrides, unreconciled submitted intents, protection counts, and
     local outcome evidence.
   - Include normal open orders and algo orders in the packet with blocking
     reasons.
   - Emit read-only flags proving no orders, cancels, env writes, systemd
     changes, profile applies, or order intents were created.

2. Add manual loss incident intake module.
   - Create `src/bfa/ops/manual_loss.py`.
   - Define schema `bfa_manual_loss_incident_v1`.
   - Validate symbol, side, leverage, entry price, exit price or liquidation
     price, stop-loss status, trigger reason, and lessons.
   - Persist incidents append-only through the existing event store, preferably
     as `risk_state` artifacts with event type `manual_loss_incident`.
   - Redact/avoid secrets and account credentials.

3. Add CLI commands.
   - Add `ops exposure-clearance`.
   - Support `--manual-exposure-symbols`, `--skip-binance`, `--target-profile`,
     and `--allow-two-positions` consistent with existing ops commands.
   - Add `ops manual-loss-record`.
   - Accept structured CLI fields plus `--db` and `--occurred-at`.
   - Return JSON only; exit non-zero only for validation or persistence errors.

4. Connect operator resume decision to clearance evidence.
   - Let `ops operator-resume-decision` accept an optional
     `--exposure-clearance-report` JSON artifact.
   - If clearance contains blocking manual/unknown/stale exposure, keep status
     `resolve_exposure`.
   - Include clearance status and blocking classifications in the decision
     packet without allowing mutation.

5. Add tests.
   - Unit-test agent-managed vs manual vs unknown exposure classification.
   - Unit-test stale-attributed behavior when local submitted intent evidence
     does not cleanly clear the active exposure.
   - Unit-test normal open orders and orphan algo orders as blockers.
   - Unit-test manual loss incident validation and append-only persistence.
   - CLI smoke-test both new ops commands and the operator decision artifact
     integration.

6. Close the phase.
   - Write Phase 56 summary and verification artifacts.
   - Update `ROADMAP.md`, `STATE.md`, `PROJECT.md`, and `REQUIREMENTS.md`.
   - Commit implementation and planning artifacts.

</tasks>
