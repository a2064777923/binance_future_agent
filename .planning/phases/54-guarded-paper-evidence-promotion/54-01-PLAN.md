# Plan 54-01: Guarded Paper Evidence Promotion

<objective>
Collect current matrix and post-change paper evidence for
`quant_setup_selective_guarded`, prove server paper collection can run without
live mutation, and keep the result fail-closed unless the evidence genuinely
passes promotion gates.
</objective>

<tasks>

1. Capture a phase evidence boundary.
   - Record an ISO timestamp before running guarded paper collection.
   - Use that timestamp as the `--since` boundary for post-change performance.

2. Rerun current matrix evidence.
   - Run `backtest matrix-suite` for `quant_setup_selective_guarded` across
     `5m,15m` and `broad,momentum,liquid`.
   - Write runtime output to `runtime/quant_setup_matrix_phase54.json`.
   - Compare verdict, total net PnL, worst drawdown, and candidate matrix count
     against Phase 50's `quant_setup_selective_guarded` evidence.

3. Run server guarded paper collection.
   - Confirm live timer/service are inactive before running.
   - Run server `ops forward-paper-run` with
     `--auto-hot-symbols --top-n 40 --interval 5m --variant quant_setup_selective_guarded`.
   - Confirm the result is paper-only and does not create live order intents.
   - Confirm live timer/service remain inactive after the run.

4. Evaluate post-change paper gates.
   - Run server `ops forward-paper-performance-check` with the Phase 54
     timestamp boundary, `quant_setup_selective_guarded`, `5m`, and the v1.24
     thresholds.
   - Accept `insufficient_evidence` or negative metrics as a valid fail-closed
     result.

5. Feed evidence back into readiness.
   - Run server `ops live-resume-readiness` with the Phase 54 matrix report path
     if available on the server, or record `matrix_report_not_provided` if only
     local matrix evidence exists.
   - Keep manual exposure symbols as `ETHUSDT`.
   - Verify live resume remains blocked unless all gates pass.

6. Write closeout artifacts.
   - Create `54-01-SUMMARY.md`.
   - Create `54-VERIFICATION.md`.
   - Run local verification commands and secret scans.

</tasks>

<verification>

- `python -m unittest tests.test_ops_forward_paper_performance tests.test_ops_live_resume_readiness`
- `python -m unittest discover -s tests`
- `git diff --check`
- Runtime artifact secret scans pass.
- Server units before and after evidence collection show live timer/service
  inactive.
- Server post-change paper performance uses the captured Phase 54 timestamp
  boundary.

</verification>

<success_criteria>

- Current guarded matrix evidence is captured and compared with Phase 50.
- Server guarded paper collection runs or produces a clear blocker without live
  timer/service/profile/order mutation.
- Post-change paper evidence has an explicit boundary and promotion verdict.
- Live resume remains fail-closed unless matrix, paper, server, exposure,
  profile, and confirmation gates all pass.

</success_criteria>
