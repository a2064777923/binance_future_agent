# Plan 63-01: Live Cycle Position Stewardship

<objective>
Make each live cycle persist active-position lifecycle decisions before new
entry candidate generation or AI calls, while keeping deterministic
auto-management disabled unless explicit env flags are enabled.
</objective>

<tasks>

1. Add explicit auto-management config.
   - Add `BFA_POSITION_AUTO_MANAGEMENT_ENABLED=false`.
   - Add `BFA_POSITION_AUTO_MANAGEMENT_MAX_ACTIONS_PER_CYCLE=1`.
   - Keep defaults dormant and validate numeric max actions as positive integer
     if the project validation pattern supports it.

2. Persist lifecycle decisions.
   - Add a lifecycle payload builder from the current position-adjustment plan.
   - Persist a `risk_state` artifact with
     `event_type=position_lifecycle_decision` before market snapshots,
     candidate generation, or AI decisions.
   - Include auto-management config state and Phase 61 diagnostics.

3. Improve run summaries.
   - Add diagnostics to `_position_adjustment_summary()`.
   - Add `position_lifecycle` event ID to `AgentRunResult.persisted`.

4. Test ordering and boundaries.
   - Add live runner tests proving lifecycle artifact exists before candidates.
   - Add tests proving manual positions are recorded as `manual_hold`.
   - Add config tests for default disabled auto-management.

5. Deploy preview-only.
   - Run local focused/full tests.
   - Deploy to server with timers paused.
   - Run server focused/full tests.
   - Let or trigger only normal timer preview behavior; do not enable
     auto-management or submit exits.

6. Close GSD phase.
   - Write summary and verification.
   - Update ROADMAP, REQUIREMENTS, STATE, and PROJECT.
   - Commit Phase 63 implementation and closeout.

</tasks>
