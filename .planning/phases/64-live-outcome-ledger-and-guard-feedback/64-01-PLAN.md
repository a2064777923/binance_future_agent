# Plan 64-01: Live Outcome Ledger And Guard Feedback

<objective>
Add a live outcome ledger command that can reconcile recent closed live trades,
summarize live performance by trade context, and emit recommendation-only guard
feedback without mutating strategy/risk configuration.
</objective>

<tasks>

1. Build live outcome ledger module.
   - Add dataclasses/report helpers in `src/bfa/ops/live_outcome_ledger.py`.
   - Load live outcomes from `outcomes`, join to `order_intents` by
     `intent_event_id`, and join to `trade_setups` / `ai_decisions` by
     symbol + decided timestamp when present.
   - Compute summary totals: outcome count, win/loss count, win rate, net PnL,
     gross profit/loss, profit factor, worst drawdown, and open/unreconciled
     submitted intent count.

2. Add optional reconciliation.
   - Let the report run `reconcile_submitted_trade_outcomes()` when requested.
   - Allow `persist_closed` only when reconciliation is requested.
   - Keep persistence limited to idempotent fills/outcomes.

3. Add attribution and guard recommendations.
   - Group outcomes by symbol, side, exit reason, setup reasons, factor names,
     factor reasons, and holding-time bucket.
   - Generate recommendation-only guard feedback for negative groups.
   - Include mutation proof with `applies_changes=false`, `raises_risk=false`,
     `places_orders=false`, `cancels_orders=false`, and `writes_env_files=false`.

4. Add CLI command.
   - Add `ops live-outcome-ledger`.
   - Flags: `--env-file`, `--db`, `--symbol`, `--since`, `--latest-limit`,
     `--min-group-outcomes`, `--reconcile`, `--persist-closed`.
   - In live/testnet reconciliation mode, use the signed Binance client. In
     read-only DB mode, do not require exchange credentials.

5. Add tests.
   - Unit-test aggregation and grouped recommendations.
   - Unit-test optional reconciliation persistence with a fake signed client.
   - CLI-test JSON shape and non-mutation proof.

6. Deploy preview-only.
   - Run local focused/full tests and `git diff --check`.
   - Deploy with timers paused.
   - Run server focused/full tests.
   - Run read-only server ledger smoke and, if safe, reconcile-only smoke that
     persists closed outcomes idempotently without orders.
   - Restore timers and verify services inactive.

7. Close GSD phase.
   - Write summary and verification.
   - Update ROADMAP, REQUIREMENTS, STATE, and PROJECT.
   - Commit implementation and closeout.

</tasks>

<success_criteria>

- Closed live outcomes can be reconciled idempotently and shown in one ledger.
- Operator can review performance by symbol, side, setup profile/reasons,
  factor evidence, exit reason, and holding behavior.
- Weak live outcome groups produce recommendation-only guard feedback.
- Ledger and recommendations cannot raise risk or mutate live env/strategy by
  themselves.
</success_criteria>
