# Plan 36-01: Indicator-Based Setup Point Logic

## Goal

Make deterministic setup scoring and point selection visibly quantitative, so
operator traces can explain why a side, entry estimate, stop, target, and size
were selected without relying on the AI layer to invent prices.

## Scope

- Add a dependency-free kline indicator helper for ATR, VWAP, EMA spread, RSI,
  support/resistance, realized volatility, momentum, and volume impulse.
- Feed those indicators into live feature extraction and the `quant_setup`
  backtest candidate builder.
- Extend setup scoring with trend structure, RSI regime, and volume impulse
  factors.
- Generate a `price_basis` payload that explains stop/target anchors and
  risk/reward geometry.
- Expose the new fields through AI compact context and read-only
  `ops trade-trace`.

## Out Of Scope

- No live order, close, risk-profile apply, live timer restore, or exchange
  mutation.
- No profitability claim. This phase makes the setup more testable and
  auditable; it does not prove edge.

## Verification

- Focused strategy, AI schema, backtest, and trace tests.
- Full local test suite.
- `git diff --check`.
