# Plan 29-01: Confirmation-Gated Risk Profile Switch

## Objective

Add an operator-safe way to preview and later apply the 30U/8x dynamic sizing
profile without hand-editing the server env file.

## Scope

- Add `ops risk-profile-plan`.
- Add `ops risk-profile-apply`.
- Support a named `30u_8x_dynamic` profile:
  - `BFA_ACCOUNT_CAPITAL_USDT=30`
  - `BFA_MAX_LEVERAGE=8`
  - `BFA_MAX_POSITION_NOTIONAL_USDT=20`
  - `BFA_MAX_RISK_PER_TRADE_USDT=0.3`
  - `BFA_MAX_DAILY_LOSS_USDT=1`
  - `BFA_MAX_OPEN_POSITIONS=1` by default
  - `BFA_DYNAMIC_POSITION_SIZING_ENABLED=true`
  - `BFA_MAX_MARGIN_PER_POSITION_USDT=3`
  - `BFA_MAX_MARGIN_FRACTION=0.08`
  - `BFA_MAX_EFFECTIVE_NOTIONAL_USDT=30`
  - `BFA_MULTI_POSITION_ENABLED=false` by default
- Optionally support a `--allow-two-positions` preview that sets
  `BFA_MAX_OPEN_POSITIONS=2` and `BFA_MULTI_POSITION_ENABLED=true`.
- Emit a confirmation token derived from the env diff.
- Refuse to apply unless:
  - `ops risk-change-check --target-leverage 8` allows the change,
  - the live service is inactive,
  - the confirmation token matches,
  - the target env path is explicitly supplied.
- Back up the env file before writing.
- Write only approved non-secret BFA risk/profile keys.

## Verification

- Unit tests for profile diff/token generation.
- CLI tests for plan output and blocked apply while risk-change readiness fails.
- Synthetic env-file apply test verifies backup creation and key preservation.
- Full local suite passes.
- Server non-trading verification runs plan/apply without token and confirms
  live env is unchanged.

## Safety Notes

This phase creates the switch mechanism but must not apply it to the live server
while HYPEUSDT remains open. A later operator-approved apply should happen only
after `risk-change-check` returns allowed.
