# Plan 19-01: 30U Higher-Leverage Trial Profile

## Objective

Configure the live system for the user's proposed 30 USDT trial while allowing a
modest increase from 3x to 5x leverage and reducing absolute loss exposure.

## Scope

- Update only server runtime risk-profile values:
  - `BFA_ACCOUNT_CAPITAL_USDT=30`
  - `BFA_MAX_LEVERAGE=5`
  - `BFA_MAX_POSITION_NOTIONAL_USDT=12`
  - `BFA_MAX_RISK_PER_TRADE_USDT=0.3`
  - `BFA_MAX_DAILY_LOSS_USDT=1`
  - `BFA_MAX_OPEN_POSITIONS=1`
- Preserve DeepSeek provider settings, Binance credentials, cross margin,
  hedge position mode, protective-order requirement, and systemd isolation.
- Capture redacted server evidence proving the active profile.
- Reconcile any already-submitted live order before re-enabling automation.
- Keep the live timer disabled if a real position is open and needs operator
  review.

## Verification

- Local config test suite still passes.
- Server health check passes after env update.
- Server live-status reports the current exchange state, including positions,
  normal open orders, and algo TP/SL orders.
- Server live-status reports no active AI backoff.
- Timer state is intentionally reviewed after reconciliation instead of assumed
  active.
- GSD requirements and roadmap reflect the active 30U trial profile.

## Safety Notes

This is not a strategy scale-up. It lowers configured capital from 100 USDT to
30 USDT, lowers max notional from 20 USDT to 12 USDT, lowers single-trade risk
from 1 USDT to 0.3 USDT, lowers daily loss from 3 USDT to 1 USDT, and limits
concurrency to one position. The higher leverage setting improves minimum-order
capital efficiency, but deterministic notional/risk caps still control exposure.
