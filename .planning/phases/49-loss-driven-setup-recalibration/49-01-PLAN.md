# Plan 49-01: Loss-Driven Setup Recalibration

<objective>
Convert current forward-paper loss attribution into a stricter deterministic
setup profile that can be tested in backtests and forward-paper without
changing live defaults.
</objective>

<tasks>

1. Extend setup-profile gates.
   - Support blocked setup reasons.
   - Support blocked negative factor names.
   - Support open-interest, liquidity, momentum, and volume-impulse minimums.

2. Add a paper/backtest-only variant.
   - Add `quant_setup_loss_recalibrated`.
   - Keep `quant_setup`, `quant_setup_selective`, and live defaults unchanged.
   - Encode current weak groups: worst symbols, short side, stop-loss-prone
     taker/volume/RSI/momentum/trend conditions.

3. Add tests.
   - Verify default setup behavior is unchanged.
   - Verify the new profile can block weak setup reasons and negative factor
     names.
   - Verify CLI/backtest/matrix recognize the new variant.

</tasks>

<verification>

- Run focused setup/backtest/CLI tests.
- Run the full unit suite.
- Run `git diff --check`.
- Run a secret scan before commit.

</verification>

  unchanged.

</success_criteria>
