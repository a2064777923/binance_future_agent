# Plan 41-01: Forward-Paper Scheduling Assets

## Goal

Add isolated systemd assets that can run the forward-paper recorder on a
schedule without enabling live automation.

## Scope

- Add `binance-futures-agent-paper.service`.
- Add `binance-futures-agent-paper.timer`.
- Extend the remote bootstrap allowlist and install step for the paper unit and
  timer.
- Update deployment documentation with manual paper-service and paper-timer
  operations.
- Add tests that confirm paper assets run `ops forward-paper-run` and do not
  run `agent run-once`.

## Out Of Scope

- No live timer restore.
- No risk profile apply.
- No exchange order, close, or position adjustment.
- No systemd auto-enable or auto-start during bootstrap.
- No claim that forward-paper permission is equivalent to live-resume
  permission.

## Verification

- Full local test suite.
- `git diff --check`.
- Secret-pattern scan over the working diff.
- Server deployment and service-state checks after commit.
