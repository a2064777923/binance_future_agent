<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| environment -> config object | Untrusted local environment values become typed runtime configuration. |
| env file -> config object | Optional local env files may contain secrets and malformed values. |
| config object -> diagnostics | Config summaries may be printed by CLI or tests only after redaction. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Spoofing | `BFA_MODE` | mitigate | Reject unknown modes and default only absent mode to `dry_run` per D-04. |
| T-01-07 | Tampering | risk limit env values | mitigate | Parse numeric risk fields and reject missing, invalid, zero, or negative values where required. |
| T-01-08 | Information Disclosure | validation result summaries | mitigate | Pass summaries through `redact_object` before exposing diagnostics per D-05. |
| T-01-09 | Elevation of Privilege | `live` mode config | mitigate | Require explicit Binance credentials, required risk caps, and kill-switch path before live config validates per D-04. |
| T-01-SC | Tampering | package installs | accept | This plan adds no external packages. |
</threat_model>
