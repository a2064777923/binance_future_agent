<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| terminal user -> CLI arguments | CLI arguments may point to env files or request validation modes. |
| config validation -> terminal output | Validation diagnostics become visible in terminal output and must be redacted. |
| deployment documentation -> server operations | Documentation guides later server changes and must preserve isolation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Information Disclosure | `src/bfa/cli.py` config-check output | mitigate | Print only validation result fields and redacted config summaries per D-05. |
| T-01-11 | Tampering | `config-check --env-file` | mitigate | Treat env-file input as local config data, validate mode and numeric fields, and avoid executing file content. |
| T-01-12 | Tampering | `docs/deployment_isolation.md` | mitigate | Document isolated paths and explicitly prohibit changes to existing server projects per D-06. |
| T-01-13 | Information Disclosure | final repository state | mitigate | Run unittest, `git diff --check`, and secret-pattern scanning before completion. |
| T-01-SC | Tampering | package installs | accept | This plan adds no external packages. |
</threat_model>
