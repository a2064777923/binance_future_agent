# Plan 26-01: Time Exit Plan

## Objective

Add a read-only plan command for positions that exceeded AI hold-time guidance,
so an operator can inspect the exact close-order shape before any future
execution-capable time-exit phase.

## Scope

- Add `ops time-exit-plan`.
- Reuse the Phase 25 hold-time check.
- Require hold-time expiry, an active matching submitted intent, a non-zero
  position amount, and confirmed algo protection before a plan is marked ready.
- Emit the planned close order side, order type, quantity, position side, and
  reduce-only flag.
- Respect hedge mode by using `positionSide` and not setting Binance
  `reduceOnly`.
- Keep the command read-only: no order placement, no order cancel, no leverage
  change, no env edit, and no timer change.
- Add unit and CLI tests.

## Verification

- Unit tests cover ready and blocked time-exit plans.
- CLI tests cover JSON output and exit code for a ready plan.
- Full local test suite passes.
- Server focused/full tests pass after deployment.
- Server live `ops time-exit-plan` reports the current BNBUSDT close plan
  without mutating exchange state.

## Safety Notes

This phase intentionally stops before automatic or operator-approved execution.
It produces a plan only. A later phase must add explicit approval, service
coordination, exchange execution, protective-order cleanup, and outcome
reconciliation before any time-exit action can be automated.
