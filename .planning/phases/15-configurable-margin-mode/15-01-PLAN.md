# Plan 15-01: Configurable Margin Mode

## Objective

Allow the live pilot to run against the current Binance Multi-Assets account by
making margin mode explicit and validated, without increasing any pilot risk cap.

## Triggering Evidence

The first valid trade decision after expanding the pilot universe reached
pre-entry margin setup, but Binance rejected `ISOLATED` margin because the live
account is in Multi-Assets mode. Phase 14 made that condition fail closed. Phase
15 adds a deliberate cross-mode path so the pilot can proceed under the account's
current margin semantics.

## Scope

- Add `BFA_MARGIN_MODE` config with accepted values:
  - `isolated`;
  - `cross`.
- Keep default `BFA_MARGIN_MODE=isolated` in code and templates.
- Map `isolated` to Binance `ISOLATED`.
- Map `cross` to Binance `CROSSED`.
- Emit a live config warning when `BFA_MARGIN_MODE=cross`.
- Preserve all existing pilot risk controls:
  - account capital 100 USDT;
  - max leverage 3x;
  - max position notional 20 USDT;
  - max risk per trade 1 USDT;
  - max daily loss 3 USDT;
  - max open positions 2;
  - required protective orders.

## Verification

- Config tests cover invalid margin mode and live cross-mode warning.
- Execution tests cover isolated default and cross mode margin setup calls.
- Full unit suite passes.
- Server env is updated explicitly to `BFA_MARGIN_MODE=cross`.
- Server health check passes with redacted config.
- Live timer is observed after deployment.

## Safety Notes

Cross margin can expose account-level collateral differently from isolated
margin. This phase does not loosen notional/risk caps, and protective orders
remain mandatory. Future risk-cap changes require separate backtest and live
evidence.
