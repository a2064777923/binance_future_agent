# Plan 32-01: Active Position Adjustment Plan

## Goal

Convert deterministic active-position review recommendations into actionable
but confirmation-gated adjustment plans so the live runner manages open
positions before searching for new entries.

## Scope

- Add read-only `ops position-adjustment-plan`.
- Add confirmation-gated `ops position-adjustment-execute`.
- Map `trail_or_reduce` to partial take-profit reduce orders.
- Map overdue or unsafe `close_review` positions to full-close reduce orders.
- Include active-position review and adjustment summaries in each live
  `agent run-once` result.
- Preserve confirmation gates for all live exchange mutation.

## Non-Goals

- Do not silently execute a live reduce order during deployment.
- Do not apply `30u_10x_multi_dynamic` automatically.
- Do not claim public Lana-style screenshots prove profitability.

## Verification

- Focused tests for position adjustment plan/execute, position review, agent
  runner, CLI, and config.
- Full local test suite.
- Server deploy under `/opt/binance-futures-agent/app`.
- Server read-only `ops position-adjustment-plan` preview.
