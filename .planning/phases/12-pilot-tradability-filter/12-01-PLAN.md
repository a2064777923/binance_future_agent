# Plan 12-01: Pilot Tradability Filter

## Objective

Prevent the 100 USDT live pilot from selecting hot symbols that cannot satisfy
current Binance USD-M minimum quantity/notional filters under the configured
`BFA_MAX_POSITION_NOTIONAL_USDT` cap.

## Scope

- Extract Binance execution filter facts from normalized `exchange_symbol`
  snapshots:
  - market lot minimum quantity;
  - market lot step size;
  - minimum notional.
- Combine those filters with the latest kline reference price to compute
  `min_executable_notional`.
- Reject candidates whose `min_executable_notional` exceeds the configured max
  position notional.
- Include `min_executable_notional` in compact AI context.
- Reject AI trade decisions whose notional is below `min_executable_notional`.
- Keep the existing 100 USDT pilot caps, leverage, kill switch, and protective
  order requirements unchanged.

## Verification

- Unit tests cover feature extraction through candidate rejection.
- Agent runner test proves an untradable hot candidate does not call AI.
- AI validation test rejects trade notional below the executable minimum.
- Full unit suite passes.
- `git diff --check` passes except Windows LF-to-CRLF warnings.
- Server deploy health check passes after implementation.

## Safety Notes

This phase intentionally reduces live-trade attempts under small caps. It does
not increase notional caps, loosen risk gates, alter API credentials, or bypass
protective-order requirements.
