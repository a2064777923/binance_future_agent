# Plan 21-01: Closed Trade Outcome Reconciliation

## Objective

Reconstruct the first completed live trade from Binance fills and persist a
net-of-commission outcome artifact.

## Scope

- Add signed `userTrades` read support.
- Add an outcome summarizer for the latest submitted local order intent.
- Add `ops trade-outcome` with optional `--persist`.
- Persist exchange fills to `fills` and round-trip summary to `outcomes`.
- Keep the workflow read-only against Binance; do not place, cancel, or modify
  orders.

## Verification

- Unit tests cover signed `userTrades`, closed round-trip PnL math, and event
  persistence.
- CLI test covers `ops trade-outcome --persist`.
- Server run reconstructs the closed ZECUSDT live trade and writes fill/outcome
  evidence to the event store.
- Server timer remains active and no exchange state is modified by the outcome
  command.

## Safety Notes

Outcome reconciliation is observational accounting. It must not become a hidden
execution path.
